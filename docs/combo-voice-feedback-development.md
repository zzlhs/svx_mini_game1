# 连续成功语音反馈开发规格

## 1. 文档状态

- 状态：待评审
- 功能范围：玩家连续成功完成合法方块区域选择时，播放英文鼓励语音
- 目标平台：Web 与微信小游戏
- 本阶段不包含：游戏代码修改、音频复制、音频转换及界面改动

## 2. 功能目标

在玩家连续正确选择方块区域时，根据当前连续成功次数播放不同等级的英文语音，增强正向反馈。语音沿用现有“声音”设置开关，不新增单独的语音开关。

## 3. 连续成功的定义

一次“成功”指：玩家结束一次拖拽后，系统确认所选矩形区域合法，并真正新增一个方块区域。

- 无论该矩形包含多少个格子，一次合法区域选择只计为 1 次成功。
- 只在结束拖拽并提交结果时计数；拖拽预览阶段不计数。
- 仅浏览或选中已经放置的区域不计数。
- 查看提示不增加次数，也不清零。
- 连击只保存在当前运行状态中，不写入关卡进度或排行榜数据；重新启动游戏后从 0 开始。

## 4. 触发规则

| 连续成功次数 | 语音池 | 播放规则 |
| --- | --- | --- |
| 1～2 次 | 无 | 只保留现有普通放置反馈音效 |
| 恰好 3 次 | `Good` | 播放 `good` |
| 恰好 5 次 | `Great`、`Nice` | 两个语音以相同概率随机播放一个 |
| 恰好 7 次 | `Amazing`、`Prefect` | 两个语音以相同概率随机播放一个 |
| 其他次数 | 无 | 只保留现有普通放置反馈音效 |

本稿默认只在恰好达到 3、5、7 次时触发一次语音；达到 8 次及以上时不重复播放 7 次档语音。发生清零事件后，玩家可以重新触发各档语音。

## 5. 连击清零规则

以下操作将连续成功次数立即清零：

1. 选错：玩家结束拖拽并提交了一个不合法的区域。
2. 撤销：玩家执行撤销操作。
3. 删除：玩家删除一个已经放置的区域；该行为与撤销同样会破坏连续成功记录。
4. 重置关卡。
5. 切换关卡，包括上一关、下一关、从关卡面板选择关卡，以及通关后的自动进入下一关。
6. 重新开始整轮游戏或重新启动游戏。

以下行为不清零：

- 使用提示。
- 只打开或关闭提示，不提交区域。
- 拖拽过程中暂时经过不合法区域，但最终提交的是合法区域。
- 取消一次尚未提交的拖拽。
- 仅选中或取消选中已经放置的区域。

## 6. 通关声音优先级

声音优先级如下：

1. 通关音效
2. 连击语音
3. 普通成功放置音效
4. 失败音效

如果一次合法放置同时完成关卡：

- 播放现有通关音效。
- 不播放本次可能触发的 `Good`、`Great`、`Nice`、`Amazing` 或 `Prefect` 语音。
- 避免连击语音与通关音效叠加。
- 通关后将连击次数清零。

非通关情况下达到连击档位时，保留现有短促的成功放置反馈，同时播放对应英文语音。若实际试听出现声音拥挤，可在开发验收时将普通放置音效改为本次不播放，但不改变连击计算规则。

## 7. 语音语言与声音设置

- 只使用本次提供的 5 个英文语音，不根据游戏界面语言切换素材。
- 中文和英文界面使用完全相同的英文语音。
- 现有“声音”开关关闭时，普通反馈音、通关音、背景音乐和新增连击语音均不播放。
- 不新增设置项，不改变现有振动规则。

## 9. 音频处理要求

批准开发后，由 WAV 生成游戏运行时使用的 MP3，源 WAV 保持不变。

- 输出格式：MP3
- 采样率：44.1 kHz
- 声道：单声道
- 建议码率：128 kbps
- 处理：裁掉不必要的首尾静音，但保留自然尾音
- 音量：5 个文件听感保持一致，避免削波
- 建议资源目录：`assets/wechat/audio/combo/`
- 微信构建产物目录：`dist-wechat/assets/wechat/audio/combo/`

构建脚本需要负责从源资源目录复制或生成运行时文件，不能让正式游戏依赖 `Downloads` 下的本机路径。

## 10. 建议的技术实现

### 10.1 游戏状态

在共享游戏逻辑中维护一个非持久化的连续成功计数，并通过一次性效果事件通知界面层播放哪一档语音。Web 与微信小游戏共用同一套计数和触发规则，避免两端行为不同。

### 10.2 成功处理顺序

一次合法区域提交后的处理顺序建议为：

1. 新增合法区域。
2. 连续成功次数加 1。
3. 判断棋盘是否已经通关。
4. 如果通关，只发出通关效果事件，不发出连击语音事件，并清零连击。
5. 如果未通关且次数为 3、5、7，发出对应连击语音事件。
6. 其他次数只发出现有普通放置效果事件。

### 10.3 随机选择

- 第 5 次：从 `Great` 和 `Nice` 中等概率选择一个。
- 第 7 次：从 `Amazing` 和 `Prefect` 中等概率选择一个。
- 每次达到档位时独立随机，不要求记录或规避上一次播放结果。

### 10.4 音频播放

- 在现有反馈音频模块中增加连击语音播放能力。
- 微信端继续使用小游戏音频上下文播放 MP3。
- Web 端加载同一组 MP3 素材。
- 同一条连击事件只能消费一次，界面重复渲染不得导致语音重复播放。
- 新的语音播放器需要在播放结束、停止或失败后释放资源。

## 11. 验收用例

| 编号 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 连续合法放置 2 次 | 不播放连击语音 |
| 2 | 连续合法放置第 3 次，且未通关 | 播放一次 `Good` |
| 3 | 继续合法放置到第 5 次，且未通关 | 随机播放一次 `Great` 或 `Nice` |
| 4 | 继续合法放置到第 7 次，且未通关 | 随机播放一次 `Amazing` 或 `Prefect` |
| 5 | 连续合法放置第 8 次 | 不重复播放第 7 次档语音 |
| 6 | 成功 2 次后提交不合法区域，再成功 2 次 | 不播放 `Good`；错误后已从 0 重新计数 |
| 7 | 成功 2 次后使用提示，再成功 1 次 | 播放 `Good` |
| 8 | 成功后执行撤销或删除，再成功 | 从 1 重新计数 |
| 9 | 成功后重置或切换关卡 | 新关卡从 0 开始计数 |
| 10 | 第 3、5 或 7 次成功同时完成关卡 | 只播放通关音效，不播放连击语音 |
| 11 | 关闭“声音”设置后达到任一档位 | 不播放连击语音 |
| 12 | 在 Web 与微信小游戏重复上述用例 | 两端触发行为一致 |
| 13 | 同一画面发生多次渲染 | 每个连击事件只播放一次 |
| 14 | 多次重新开始连击并到达第 5、7 次 | 两个随机池中的素材均能被播放 |

## 12. 评审时需要确认的默认项

本稿已经按以下默认解释写入开发范围，评审时如无修改即可直接执行：

1. “一次成功”按一次合法矩形区域提交计算，而不是按矩形包含的格子数计算。
2. 只在恰好 3、5、7 次时播放语音，8 次以上不继续重复。
3. 删除已放置区域与撤销一样会清零。
4. 非通关的连击档位保留普通放置反馈音，并叠加英文语音。
5. `prefect.wav` 暂按现有拼写映射；是否修正为 `perfect` 可在正式开发前确认。
